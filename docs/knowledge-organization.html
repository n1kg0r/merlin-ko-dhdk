Knowledge Organization

Conceptual Map

The Conceptual Map represents the first step in the knowledge organization of the domain. It shows the data related to the items and the domain in a graphical display using natural language labels to describe the entities and the relationships among them.

E/R Model

The Entity-Relationship Model represents a translation of the Conceptual Map to a higher level of abstract formalism; it focuses on the entities, relationships and their attributes. This abstraction was obtained by expressing the data with more general concepts.

After the identification of the proper metadata standards for each item, the following step is the metadata alignment. This procedure allows us to identify how each descriptive feature of the items as it is defined by each standard. The Dublin Core (DC) is used as a guide for this process since it is one of the most generic standards. The properties are divided to better represent the different pieces of information related to people (who), places (where), dates (when), subjects/concepts (what).

Theoretical model

In the Theoretical Model the original data presented in the Conceptual Map are enriched with new properties, to express the four categories pointed out in the previous step (people, places, dates, subjects). The data are gathered from other sources.

Enriched E/R

The new real data gathered in the Theoretical model are expressed in this step in a more abstract way, resulting in an enriched version of the previous E/R Model.

The Conceptual model

The Conceptual Model is a formal representation of the theoretical model, where entities and relationships are expressed through ontologies labels. To create such a model, schemas, vocabularies and ontologies are used. To produce a graphical representation of this formalisation the Graphical Framework for Owl Ontologies, Graffoo, has been used.

Different ontologies are used to describe each concept in the most accurate way. The ontologies used are:

  • CIDOC-CRM: is a good trial of harmonisation of information coming from different domains, even if it was first intended for describing museum objects.
  • DBPEDIA ONTOLOGY: was particularly useful for expressing the difference between a real person and a fictional character (as the FOAF class Person refers only to real people).
  • DC TERMS: extremely useful for the property dcterms:alternative which allows to describe an alternative title (as in the case of the Merlijn printing)
  • FABIO: describes accurately the classes of the bibliographic material and the properties related to the activity of publishing, that are useful to describe the library domain.
  • FOAF: provides specific labels useful to describe properties related to people.
  • OWL: is used for the property isPartOf.
  • SCHEMA: applied to all items which did not necessarily belong in the LAM domain (movie and soundtrack). It was also adopted to describe persons.
  • XSD: is used to assign data types.
